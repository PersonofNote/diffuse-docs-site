npm install @personofnote/diffuse

The Problem

Breaking changes slip through code reviews and tests, causing production issues that could have been caught earlier.

💥

Silent Breaking Changes

Removed props, narrowed types, and interface changes that break downstream code

🤔

Reasonable-Looking Code

Code passes tests, looks reasonable, and works within ticket scope but breaks somewhere else

⏰

Late Discovery

Issues found in QA or production when they're expensive to fix

How Diffuse Works

1

Install & Configure

Add Diffuse to your project via npm or use it as a GitHub Action in your CI/CD pipeline

2

Analyze Changes

Diffuse analyzes your PR diffs and identifies potentially risky changes to your codebase

3

Get Risk Score

Receive a clear risk assessment with specific warnings about breaking changes

4

Fix Before Merge

Address issues early when they're easy and cheap to fix

Key Features

🎯

Smart Change Detection

Automatically flags removed props, narrowed types, untested new exports, and other risky patterns that might break existing code.

📊

Blast Radius Scoring

Get an instant risk assessment score so you know how carefully to review and test a PR before merging.

⚙️

CLI + GitHub Action

Use locally during development or integrate seamlessly into your CI/CD pipeline for automatic checks.
